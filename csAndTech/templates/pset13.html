Крестики-нолики

Используя минимакс алгоритм, создать ИИ для оптимальной игры в крестики-нолики.

Начало

  • Загрузить дистрибутив https://vadimgb.github.io/data/tictactoe.zip.
  • Разархивируйте, перейдите в директорию, выполните pip3 install -r requirements.txt, для установки пакета pygame.

Пояснение

В проект входит два основных файла: runner.py и tictactoe.py. tictactoe.py будет содержать интеллектуального агента, вычисляющего оптимальные ходы. runner.py был создан для вас, и содержит все необходимое для работы графического интерфейса игры. Как только вы дополните tictactoe.py всем необходимым, вы сможете запустить игру python3 runner.py, для игры против ИИ.

Откройте файл tictactoe.py. В начале создаются переменные: X, O, EMPTY, для представления возможных ходов на поле.

Функция начальное_состояние возвращает начальное состояние поля. Поле представлено списком из трёх списков (представляющих строки поля), каждый внутренний список содержит три элемента, принимающих значения X, O или EMPTY. Далее следуют функции, которые вы должны реализовать.

Детализация

Завершите определение функций игрок, действия, результат, победитель, конец, оценка и minimax.

  • Функция игрок для заданного состояние поля возвращает чей ход (X или О).
    • Первый ход делает X. Далее ходы чередуются.
    • В конечном состоянии поля, функция может возвращать любое значение (т.е. игра уже законченна).
  • Функция действия возвращает список возможных действий в данном состоянии поля.
    • Действие представляется кортежем (i, j), где i - строка (значения от 0 до 2) и j - ячейка в строке. Ячейка для хода.
    • Ходы делаются в ячейки, в которых ещё нет X или O.
    • Для конечного состояния, функция может возвращать любое значение.
  • Функция результат для заданного состояния поля и действия возвращает новое состояние поля, без модификации вводимого.
    • Если действие недопустимо, то программа raise ValueError.
    • Результирующее состояние получается из вводимого состояния, выполнением хода очередным игроком в ячейку, указанную в действии.
    • Важно, что выводимое состояние должно быть глубокой копией вводимого состояние, т.е. надо использовать copy.deepcopy функцию. Просто изменение ячейки, в водимом состоянии доски, неправильно.
  • Функция победитель для заданного состояния поля, возвращает победителя, если он есть.
    • Если X игрок победил, возвращает X. Если O, то возвращает O.
    • Побеждает тот, кто смог выполнить три хода, в строке или столбце, или на диагонали.
    • Если в заданном состоянии нет победителя (игра ещё не законченна, или ничья), функция должна возвратить None.
